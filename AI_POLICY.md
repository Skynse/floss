I took a look at the Godot issue list and you know what I saw? People creating clean, streamlined PRs one by one. There was no rush to get some feature implemented quickly, so why should there be any rush to get it implemented quickly here. No AI.

This project did utilize AI partially for code directory restructuring and test file creation (most of the work went there). I think those are valid use cases. AI code still writes slop in 2026, and this is just a limitation of the attention mechanism on large codebases. For small things, it works well, but you can't rely on it for something as large as this without having human review and several subagents working on smaller parts together.

I don't shell money out for subagents and AI doesn't mass mutate this codebase. I like to keep things simple and streamlined. People wrote code this way before, and still do.
