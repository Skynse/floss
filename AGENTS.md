# Rules

DO NOT RELY ON YOUR OWN KNOWLEDGE AT ALL
IF REFERENCING A LIBRARY OR CODEBASE, USE THE LATEST VERSION
iF REFERENCE A CODEBASE, PUT THE KNOWLEDGE INTO A MARKDOWN FILE, AND REFERENCE THAT MARKDOWN FILE BEFORE WRITING CODE
IF ASKED TO IMPLEMENT A FEATURE BASED ON A SPECIFICATION, OR OTHER CODEBASE PROVIDED, RESEARCH THE CODEBASE, WRITE AN MD FILE, THEN REFERENCE THAT MD FILE BEFORE WRITING CODE

NEVER USE YOUR OWN KNOWLEDGE, FOR ANYTHING AT ALL.

DO NOT GO FOR THE EASIEST SOLUTION, USE THE BEST AND MOST EFFICIENT APPROACH
AVOID THOUGHT PROCESSES LIKE, "ACTUALLY, THE SIMPLEST APPROACH", AND ONLY MAKE INFORMED, THOUGHTFUL DECISIONS, NO MATTER THE COMPLEXITY
ASK QUESTIONS INSTEAD OF OVERTHINKING

IF WRITING CODE BASED ON OTHER CODE, ALWAYS REFER BACK TO THE REFERENCED CODE TO CONFIRM THE CHANGES ARE CORRECT
IF MAKING BREAKING CHANGES, SOME TESTS WILL NEED TO BE REWRITTEN OR UPDATED. DO NOT MORPH CODE JUST SO THAT TESTS PASS

WHEN REFERENCING OTHER CODE, WRITE DOWN THE LOCATION OF THE RELEVANT FILES, AND SNIPPETS OF THE FUNCTIONS FOR RE-USE
IF THE USER COMPLAINS ABOUT PERFORMANCE AND THE APP IS RUNNING, PROFILE THE APP TO SEE WHERE THE ISSUE IS

NEVER GIT REVERT WITHOUT CONSENTs
